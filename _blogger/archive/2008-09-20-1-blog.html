--- title: Using sqlite as a production DB tags: layout: post blogger_date: 9/20/2008 10:03:00 PM --- I wanted to use sqlite3 as my production DB for a little web app that I am working on.  Wasn't sure exactly what was going to be needed, but turns out if you add the following to your cap recipie you are all set.  Shiny!   task :after_update_code do      run "ln -s #{deploy_to}/#{shared_dir}/db/production.sqlite3 #{current_release}/db/production.sqlite3"    end  You might notice I am using bort as the starting point for cookbook. They really have done everything I need, in the way I need. Since I am using it with Dreamhost (read: Passenger) and git I really didn't need to undo ANYTHING.
